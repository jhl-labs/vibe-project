## 코딩 컨벤션

### 일반 규칙

- 모든 코드는 영어로 작성 (주석 포함)
- 커밋 메시지는 [Conventional Commits](https://www.conventionalcommits.org/) 형식 사용
- PR은 하나의 논리적 변경만 포함

### 네이밍 규칙

- **파일명**: kebab-case (`user-service.ts`)
- **클래스/컴포넌트**: PascalCase (`UserService`)
- **함수/변수**: camelCase (`getUserById`)
- **상수**: UPPER_SNAKE_CASE (`MAX_RETRY_COUNT`)
- **타입/인터페이스**: PascalCase with prefix (`IUserService`, `TUserData`)

### 코드 스타일

- 들여쓰기: 2 spaces (또는 프로젝트 설정에 따름)
- 최대 줄 길이: 100자
- 세미콜론: `<사용/미사용>`
- 따옴표: `<작은/큰따옴표>`

## 아키텍처

### 핵심 원칙

1. **단일 책임 원칙 (SRP)**: 각 모듈은 하나의 책임만
2. **의존성 역전 원칙 (DIP)**: 추상화에 의존
3. **레이어드 아키텍처**: 명확한 계층 분리

### 디렉토리 구조 규칙

```
src/
├── domain/       # 비즈니스 로직 (순수 함수)
├── application/  # 유스케이스
├── infrastructure/ # 외부 서비스 연동
└── presentation/ # API/UI 레이어
```

## 보안 가이드라인

### 금지 사항

- 하드코딩된 시크릿 (API 키, 비밀번호 등)
- SQL 인젝션 가능한 쿼리
- eval() 또는 동적 코드 실행
- 검증되지 않은 사용자 입력 사용

### 필수 사항

- 모든 사용자 입력 검증
- 환경 변수를 통한 설정 관리
- HTTPS 통신
- 적절한 에러 처리 (민감 정보 노출 방지)
